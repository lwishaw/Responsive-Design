Strategies for improving load time of web sites

Website visitors may enjoy a well-designed website, but if the site is slow to load — they most likely will look elsewhere. In a study done in 2011 by Akamai.com and Gomez.com, about half of web users expect a site to load in 2 seconds or less. If it isn’t loaded within 3 seconds, those users tend to abandon the site.

There are several ways to improve a page’s load speed.

Code the site for mobile-first quality and speed

It’s critical to ensure a website is mobile first. The industry has shifted in its thinking and the strategy should be to code for mobile users first and then progressively enhance the experience for tablets and desktops. To accomplish this, use Google’s Mobile Website Auditor to test the site. The test will show your website’s score by a selection of usability criteria, like viewpoint configuration, plugins, proximity of clickable elements and text readability.

Optimize user-experience design by thinking about the workflow strategy, SEO optimization, voice search and make copy readable on all devices. Platforms such as Wix, Squarespace and Google Sites have changed their setups to think in mobile first.

Reduce the number of images used

Design has shifted to a more minimal, crisper look compared to the earlier days of web design. Rethink the background image, especially if the file size is big. This may not only help reduce image HTTP requests but may improve the user experience by removing distracting images that don’t correspond to the written content plan.

Reduce the size of the images

Make sure the images are appropriately-sized. The quality of a web image can still hold up at smaller sizes versus those needed for traditional print design. New formats such as WebP, created by Google developers, provides superior lossless compression for images on the web. WebP lossless images are 26 percent smaller in size compared to pngs. WebP lossy images are 25-34 percent smaller than comparable JPEG images at equivalent SSIM quality index. To download and start using WebP, see tinyurl.com/afjrref.

Hero images are popular design options these days. Optimization software such as Compressor.io or Image Optimizer can help keep the images below 150 KB, nothing above 1920 pixels in width, at an average quality of 72 dpi quality level. Any larger than this and images will load late after the page renders.

Another option is to combine images into CSS sprites to reduce roundtrips. This combines commonly used images into a single image file, thus reducing the number of HTTP request that are required to download the webpage. This feature is implemented through proper use of the CSS background-image and background-position properties.

And set a container property with max width not to surpass 90 percent of boundary width by using this formula: .container {width: 800px; max-width: 90%;}. Finally, make sure the images auto-resize to the max width: img {max-width: 100%; height: auto;}

Minimize HTTP Requests

Reduce the number of HTTP (Hypertext Transfer Protocol) requests (images, stylesheets, scripts, Flash, etc.) from a web server is the key to faster pages. According to Yahoo, these requests tend to take up about 80 percent of a page’s load time. The browser also limits requests between 4-8 simultaneous connections per domain which means loading 30-plus assets at once is not an option.

To do so, combine the CSS/JS files into one larger file, which will allow the browser to search less and create a faster load time. Queries can help load only what’s needed. Use conditional statements in the JavaScript file to run a certain script on mobile only or if images only need to load on a desktop. It allows the browser to load only what is needed versus having it try to load unnecessary scripts or images for viewports or certain devices. This helps increase speed load of the page.
